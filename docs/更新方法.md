# Google Play リリース手順

## 前提ファイル

| ファイル | 場所 | 備考 |
|---|---|---|
| キーストア | `android/app/upload-keystore.jks` | リポジトリ外で必ずバックアップ |
| 署名設定 | `android/key.properties` | .gitignore 済み。紛失注意 |

---

## 1. バージョン番号を上げる

`pubspec.yaml` の `version` 行を編集する。

```
version: 1.0.0+3
         ^^^^^  ^
         |      versionCode（Google Play の内部バージョン番号、毎回 +1 必須）
         versionName（ユーザーに表示される番号）
```

例：バグ修正リリースの場合
```yaml
version: 1.0.1+4
```

> **ルール**: versionCode は前回より必ず大きい値にする。同じ値はアップロード不可。

---

## 2. App Bundle をビルドする

プロジェクトルートで実行：

```bash
flutter build appbundle --release
```

完了すると以下に生成される：

```
build/app/outputs/bundle/release/app-release.aab
```

ビルドエラーが出た場合は先に `flutter clean` を実行してから再試行する。

---

## 3. Google Play Console にアップロードする

1. [Google Play Console](https://play.google.com/console) を開く
2. **WebVoice** アプリを選択
3. 左メニュー → **テストとリリース** → **製品版**
4. 右上の **新しいリリースを作成** をクリック
5. `app-release.aab` をドラッグ&ドロップでアップロード
6. **リリース名**（例: `1.0.1(4)`）と**リリースノート**（日本語）を記入する
7. 右下の **次へ** → **保存**
8. **公開の概要** に移動して **変更を審査に送信** をクリック

---

## 4. 実機（USB接続）でリリース版を動作確認する

```bash
flutter install --release
```

デバッグ版と署名が異なるためインストールに失敗する場合は、先にアンインストールする：

```bash
adb uninstall com.shonancraft.webvoice
flutter install --release
```

---

## 5. リリースノートの書き方（例）

```
・読み上げカードが音声より速く進む問題を修正しました。
・ユーザーガイドのアイコンをアプリと統一しました。
```
