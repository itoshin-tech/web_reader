# 読み上げカード生成プロセスとエラーハンドリング

## 全体フロー

```
[記事選択 or URL共有]
        ↓
   ReaderNotifier.build()
        ↓
   _init()  ← TTS初期化
        ↓
   ┌─────────────────────────┐
   │ URLモード               │ RSSモード
   │ _extractFromUrl()       │ _extractAndProcessArticle()
   └─────────────────────────┘
        ↓
   ArticleExtractor.extract()   ← HTTP取得 + HTML解析
        ↓
   _toParagraphData()           ← 300文字制限・ParagraphData変換
        ↓
   state に最大50件を反映        ← UI即時表示（翻訳前）
        ↓
   _translateAll()              ← 非同期翻訳（600ms間隔）
        ↓
   各カードを isTranslated:true に更新
```

---

## フェーズ1：記事取得（ArticleExtractor.extract）

### 処理内容

| ステップ | 処理 |
|----------|------|
| HTTP GET | モバイルUA + Referer付きでHTTPリクエスト（タイムアウト5秒） |
| デコード | `utf8.decode(bodyBytes, allowMalformed: true)` でUTF-8強制デコード |
| DOM解析 | `script / style / nav / footer / header / aside` を事前除去 |
| タイトル抽出 | `og:title` → `<title>` の優先順で取得 |
| 本文抽出 | `<article>` → `<main>` → `.article` → `.content` → `<body>` の優先順で `h2〜h6, p` タグを収集 |
| フィルタリング | ナビゲーション要素（リンク比率 > 85%）・広告文言を除外 |
| 文分割 | 日本語（。！？）・英語（.!? + 大文字始まり）で分割、略語（Mr. / U.S. 等）は分割しない |
| 言語判定 | ひらがな・カタカナが全文字の5%超 → 日本語と判定 |

### エラーと対処

| エラー | 発生条件 | 対処 |
|--------|----------|------|
| HTTP タイムアウト | 5秒以内に応答なし | `TimeoutException` スロー → 上位でキャッチ |
| HTTP 非200 | 403 / 404 / 503 等 | `Exception('サイトの情報が読み込めませんでした')` スロー |
| 文字化け | latin1 エンコードページ | `allowMalformed: true` で置換文字（?）に代替、処理続行 |
| 本文抽出ゼロ件 | JS依存サイト・特殊構造 | `'本文を抽出できませんでした。'` の1段落を返す |
| DOM解析例外 | 極端に壊れたHTML | `html` パッケージが寛容解析するため実質スローされない |

## フェーズ2：ParagraphData変換（_toParagraphData）

### 処理内容

- 300文字を超える段落を先頭300文字に切り詰め
- `isTruncated: true` フラグを立てる（UIで省略通知を表示）
- 英語記事 → `english` フィールドにセット、`japanese` は空
- 日本語記事 → `japanese` フィールドにセット、`english` は空

### エラーと対処

| エラー | 発生条件 | 対処 |
|--------|----------|------|
| なし | Dart文字列操作のみのため例外は発生しない | — |

---

## フェーズ3：ステート反映（_extractFromUrl / _extractAndProcessArticle）

### 処理内容

- 段落数が50件を超える場合、先頭50件のみ `state.paragraphs` に反映
- 残りは `_pendingParagraphs` に保持し `hasMoreContent: true` をセット
- UIはこの時点で翻訳前カードを即時表示（`isTranslated: false` → グレー表示）
- 自動再生フラグがONなら `_playCurrent()` を同時呼び出し

### エラーと対処

| エラー | 発生条件 | 対処 |
|--------|----------|------|
| タイムアウト | `ArticleExtractor.extract` が5秒超 | `errorMessage` に「5秒以内に取得できませんでした」をセット |
| その他例外 | ネットワーク断・URL不正など | `errorMessage` に `'エラー: $e'` をセット |
| キャンセル済み | タブ切替などで `_isCanceled = true` | 取得完了後に検知して `return`（ステート更新せず破棄） |
| URL変更 | 取得中に別記事へ遷移 | `state.url != url` で検知して `return`（同上） |

---

## フェーズ4：翻訳（_translateAll）

### 処理内容

- 現在の読み上げ位置から順番に翻訳（先頭から翻訳すると現在位置の翻訳が遅れるため）
- 1段落ごとに翻訳APIを呼び出し、完了後600msのインターバルを置く
- 翻訳完了後は `isTranslated: true` に更新し `isTruncated` フラグを引き継ぐ
- 翻訳待ちで再生が止まっていた場合（`_waitingForTranslation`）は翻訳完了後に再生を再開

### 翻訳エンジン（_translateText）

```
1. DeepL APIキー設定済み → DeepL APIへ POST（タイムアウト10秒）
      ↓ 失敗（例外）またはステータス非200
2. Google翻訳（translator パッケージ）へフォールバック（タイムアウト10秒）
```

### エラーと対処

| エラー | 発生条件 | 対処 |
|--------|----------|------|
| DeepL タイムアウト / 例外 | API障害・ネットワーク断 | `catch(_){}` で握り潰してGoogle翻訳へフォールバック |
| DeepL 非200レスポンス | APIキー誤り・クォータ超過 | 例外スローせず通過 → Google翻訳へフォールバック |
| Google翻訳 タイムアウト | 10秒超 | `TimeoutException` スロー → `_translateAll` の `catch(e)` へ |
| Google翻訳 その他例外 | レート制限・パースエラー等 | 同上 |
| 翻訳失敗（catch到達） | 上記すべてが失敗した場合 | `(翻訳エラー)` / `(Translation failed)` を表示し `isTranslated: true` にして次の段落へ続行 |
| URL変更（記事切替） | ループ先頭で毎回チェック | `state.url != originalUrl` で検知し翻訳ループを中断（`return`） |
| 段落数変化（loadMore） | `loadMore()` 呼び出し時 | `i >= state.paragraphs.length` で検知し中断（`return`） |

## 既知のリスクと未対処項目

### ① 翻訳エラー後にインターバルがない

エラー時は `await Future.delayed(600ms)` を通らずに次の段落へ即進む。
レート制限エラーが連続した場合、ループが一時的に高速化しUIの更新が頻発する（点滅に見える原因候補）。

**暫定対処案：** エラーブロックにも `await Future.delayed` を追加

### ② `_translateAll` の2重起動防止がない

`_extractFromUrl` と `loadMore()` の両方が `_translateAll` を呼ぶ。
`loadMore()` 呼び出し直前に前の翻訳ループがまだ動いている場合、2つのループが同じ `state.paragraphs` を交互に書き換える競合が起きうる（点滅に見える原因候補）。

**暫定対処案：** `_isTranslating` フラグによる排他制御

### ③ Google翻訳のレート制限

Google翻訳パッケージは非公式APIを使用しており、短時間に多数のリクエストを送るとレート制限（429）が発生する可能性がある。600msインターバルはその対策だが、50段落の記事では約30秒かかる。

## 各段階でのUI表示状態

| 状態 | 英語テキスト | 日本語テキスト |
|------|-------------|--------------|
| 取得中 | ローディング表示 | ローディング表示 |
| 取得完了・翻訳前 | 本文テキスト（黒） | 「翻訳中...」（グレー） |
| 翻訳完了 | 本文テキスト（黒） | 翻訳テキスト（青灰色） |
| 翻訳失敗 | 本文テキスト（黒） | 「(翻訳エラー)」（青灰色） |
| 300文字超え | 本文テキスト（先頭300文字） | 翻訳テキスト ＋ 省略通知（イタリック小文字） |
| 50件超え | 50件目まで表示 | 「続きを読む」ボタン |
